GUIA - HISTORICO DA TORRE DE CONTROLE

Objetivo
Guardar snapshots periodicos da Scania/rFMS para calcular os indicadores reais
de 24h, 3 dias, 7 dias, 15 dias, 30 dias e 60 dias no dashboard.

1. Criar a tabela no Supabase

No Supabase:

1. Abra SQL Editor.
2. Clique em New query.
3. Cole o conteudo do arquivo:
   supabase/torre_historico.sql
4. Clique em Run.

A tabela criada sera:

public.torre_snapshots

Ela guarda:
- data/hora da coleta
- empresa
- VIN
- placa/nome do veiculo
- odometro
- combustivel total usado
- velocidade
- diesel %
- ARLA %
- localizacao
- alertas

2. Criar a chave secreta para a funcao Netlify

No Supabase:

1. Abra Project Settings.
2. Abra API Keys.
3. Copie uma Secret key ou Service role key.

No Netlify:

1. Abra Project configuration.
2. Abra Environment variables.
3. Cadastre:

SUPABASE_SERVICE_ROLE_KEY = cole a chave secreta do Supabase

Opcional, se um dia mudar o projeto Supabase:

SUPABASE_URL = https://txwsqgojfaiivpffgclw.supabase.co

Opcional, para mudar o tempo de retencao:

TORRE_HISTORY_RETENTION_DAYS = 90

Importante:
Nunca coloque essa chave dentro do app, dashboard HTML ou GitHub publico.
Ela deve ficar somente no Netlify.

3. Coleta automatica

Foi criada a funcao:

netlify/functions/torre-coletor.mjs

Ela roda automaticamente a cada 30 minutos pelo Netlify e chama a Scania/rFMS.
Cada execucao salva um retrato da frota na tabela torre_snapshots.

Tambem existe uma rota manual:

/api/torre-coletor

Ela serve para testar se a coleta esta funcionando.

4. Como os periodos sao calculados

O sistema pega o primeiro e o ultimo snapshot dentro do periodo:

KM do periodo = odometro final - odometro inicial
Litros do periodo = combustivel total final - combustivel total inicial
KM/L = KM do periodo / litros do periodo
Velocidade maxima = maior velocidade registrada no periodo
% acima de 100 = leituras acima de 100 / leituras em movimento

5. Depois do deploy

1. Faca deploy no Netlify.
2. Aguarde a primeira execucao automatica ou abra /api/torre-coletor.
3. Depois de algumas horas, o periodo 24h comeca a ficar mais completo.
4. Depois de 3 dias, o periodo 3d passa a ter base real.
5. Depois de 7, 15, 30 e 60 dias, os respectivos graficos ficam completos.

6. Observacao importante

O historico nao aparece completo no primeiro dia.
Ele passa a existir a partir do momento em que a coleta automatica comeca.

Por padrao o sistema mantem 90 dias de snapshots e apaga registros mais antigos.

Isso evita estourar o limite de requisicoes da Scania, porque o dashboard passa
a consultar nossa base historica em vez de pedir tudo direto da Scania a cada acesso.
