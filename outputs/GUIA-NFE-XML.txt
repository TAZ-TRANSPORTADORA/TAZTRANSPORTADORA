GUIA - NF-e / XML NO APP TAZ

O que o app faz
- Em Abastecimento e Carregamento, cada nota fiscal tem:
  - botao para ler a chave pela camera;
  - botao Buscar dados da NF;
  - botao Importar XML.

Novo modo recomendado: SEFAZ com certificado digital
O app pode buscar o XML pela Distribuicao DF-e da SEFAZ usando os certificados
digitais A1 dos CNPJs das empresas.

Importante
- Use certificado A1 em arquivo .pfx ou .p12.
- Nao envie certificado para o GitHub.
- Nao coloque certificado dentro do app.
- Certificado A3, token ou cartao, normalmente nao funciona em Netlify.

Por que nao colocar o certificado em Environment variables?
O Netlify/AWS Lambda tem limite total de variaveis de ambiente. Certificados
em Base64 passam desse limite e fazem o deploy falhar.

Por isso, o projeto agora usa:
- Environment variables apenas para configuracoes pequenas e senhas.
- Netlify Blobs para guardar o conteudo Base64 do certificado.

1. Variaveis que DEVEM ficar no Netlify
Cadastre ou mantenha apenas estas variaveis pequenas:

NFE_SEFAZ_CONFIG_JSON
NFE_CERT_TAZ_PASSWORD
NFE_CERT_PRATAZIL_PASSWORD
NFE_CERT_RILUCA_PASSWORD

Exemplo de NFE_SEFAZ_CONFIG_JSON:
[
  {
    "empresa": "TAZ",
    "cnpj": "00000000000000",
    "uf": "SP",
    "ativo": true
  },
  {
    "empresa": "Pratazil",
    "cnpj": "00000000000000",
    "uf": "SP",
    "ativo": true
  },
  {
    "empresa": "Riluca",
    "cnpj": "00000000000000",
    "uf": "SP",
    "ativo": true
  }
]

Troque os CNPJs e UFs pelos dados reais de cada empresa.

2. Variaveis que DEVEM ser removidas do Netlify
Remova todas as variaveis grandes de certificado, como:

NFE_CERT_TAZ_PFX_BASE64
NFE_CERT_TAZ_PFX_BASE64_1
NFE_CERT_TAZ_PFX_BASE64_2
NFE_CERT_PRATAZIL_PFX_BASE64
NFE_CERT_PRATAZIL_PFX_BASE64_1
NFE_CERT_PRATAZIL_PFX_BASE64_2
NFE_CERT_RILUCA_PFX_BASE64
NFE_CERT_RILUCA_PFX_BASE64_1
NFE_CERT_RILUCA_PFX_BASE64_2

Se essas variaveis continuarem no Netlify, o deploy pode falhar com erro de
limite de 4KB da AWS Lambda.

3. Fazer o deploy
Depois de remover as variaveis grandes, suba este pacote no GitHub e rode:

Deploys > Trigger deploy > Deploy project without cache

4. Enviar os certificados para o Netlify Blobs
Depois que o deploy estiver online, envie cada certificado pelo PowerShell.

Exemplo TAZ:
$certificado = 'C:\Users\Neo Loc\Downloads\2025 TAZ TRANSPORTADORA DE PRODUTOS RODOVIARIOS LTDA_52008183000195.pfx'
$base64 = [Convert]::ToBase64String([IO.File]::ReadAllBytes($certificado))
$body = @{ empresa = 'TAZ'; pfxBase64 = $base64 } | ConvertTo-Json -Compress
Invoke-RestMethod -Method Post -Uri 'https://tazmotoristas.netlify.app/api/nfe-certificados' -Headers @{ 'x-taz-admin-key' = 'SEU_CODIGO_ADMINISTRATIVO' } -ContentType 'application/json' -Body $body

Exemplo Pratazil:
$certificado = 'C:\Users\Neo Loc\Downloads\PRATAZIL HORTIGRANJEIROS LTDA.pfx'
$base64 = [Convert]::ToBase64String([IO.File]::ReadAllBytes($certificado))
$body = @{ empresa = 'Pratazil'; pfxBase64 = $base64 } | ConvertTo-Json -Compress

Exemplo Riluca:
$certificado = 'C:\Users\Neo Loc\Downloads\RILUCA COMÉRCIO DE FRUTAS LTDA.pfx'

Troque SEU_CODIGO_ADMINISTRATIVO pelo valor da variavel TAZ_ADMIN_KEY.

5. Fluxo de busca
1. O motorista le ou digita a chave da NF-e.
2. O app envia a chave para a funcao segura do Netlify.
3. A funcao consulta a SEFAZ usando o certificado da empresa.
4. Se a SEFAZ retornar XML completo, o app preenche os campos.
5. Se nada retornar XML, o motorista ainda pode usar Importar XML manualmente.

Observacoes importantes da SEFAZ
- A NF-e precisa estar autorizada para o CNPJ consultado.
- Se o CNPJ nao for emitente, destinatario, transportador ou autorizado no XML,
  a SEFAZ pode nao liberar o XML completo.
- Em alguns casos a SEFAZ retorna apenas o resumo da NF-e. Para XML completo,
  pode ser necessario manifestacao ou autorizacao do CNPJ na nota.
- Consultas repetidas em excesso podem gerar bloqueio temporario por consumo indevido.

Opcao alternativa: Meu Danfe
Se quiser manter o Meu Danfe como plano B, mantenha tambem:

MEUDANFE_API_URL
MEUDANFE_API_KEY

Exemplo de URL:
https://api.meudanfe.com.br/v2/fd/get/da/{Chave-Acesso}

Dados aproveitados do XML
- Numero da NF-e
- Serie
- Data de emissao
- Valor total
- Emitente e CNPJ
- Destinatario
- Origem e destino
- Transportadora
- Placa
- Peso bruto, peso liquido e volumes
- Produtos da nota

No abastecimento, quando o XML informa Diesel ou Arla, o app tenta preencher
automaticamente o tipo de combustivel, litros e valor.

No carregamento, o app tenta preencher a quantidade usando peso liquido,
peso bruto, soma dos produtos ou volumes, nessa ordem.
